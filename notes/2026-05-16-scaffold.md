# 2026-05-16 — Scaffold + initial reconnaissance

## What Normies is
- 10,000 fully on-chain generative NFTs on Ethereum mainnet.
- Each token = 40x40 monochrome bitmap (1600-char string), traits derived deterministically.
- Render path: bitmap → on-chain SVG; PNG also exposed via API.
- ERC721C standard, ERC-2981 royalties, SSTORE2 storage optimization, encrypted pre-reveal.
- Canvas-edit system: holders can XOR transforms into their Normie; tracked as version history. Edits cost "action points" earned by burning other Normies.
- ERC-8004 agent layer: each Normie has a live-generated persona (deterministic from immutable traits + current canvas state), an A2A card, and an entry in a paginated agent registry.

## Stack (from repo ygtdmn/normies)
- Contracts: Solidity / Foundry / ERC721C.
- API: Hono + viem + LRU cache + rate limit; pnpm for the API server, bun for tooling.
- Rate limit: 60 req/min sliding window per IP. Headers: X-RateLimit-Limit / X-RateLimit-Remaining / Retry-After.
- Auth: none — public API.

## Endpoint families to survey (one per fire)
1. `/normie/{id}/pixels` — 1600-char bitmap.
2. `/normie/{id}/traits` — trait JSON.
3. `/normie/{id}/image.svg` and `/image.png`.
4. `/normie/{id}/owner` — token owner.
5. `/holders/{address}` — owned tokens by address.
6. `/history/normie/{id}/versions` — canvas version history.
7. burn records + action points endpoints.
8. canvas transform / diff / delegation endpoints.
9. `/agents/info/{tokenId}` — live persona.
10. `/agents/binding/{tokenId}` — ERC-8004 lookup.
11. `/agents/list` — paginated registry.

## Open questions to answer over time
- Are personas seeded with a system prompt suitable for direct use, or do we need to wrap?
- Does the A2A card expose skills tied to traits, or generic?
- What does ERC-8004 binding actually attest to on-chain?
- Can a wallet-bound Normie be registered as a new agent in the network without on-chain writes from us?
- How do canvas transforms change persona deterministically — same input, same persona, but does "level-up" shift personality vector or just narrative wrapping?

## Build candidates (later phases)
- A2A card mirror: serve our owned Normie's A2A card from a stable URL we control.
- Persona-aware reply skill: use the live persona as the system prompt for a small replier bound to that Normie.
- Canvas diff visualizer: render before/after for each version, compute pixel-delta stats.
- Botchan integration: post as the owned Normie on net protocol, signed by the bound wallet.

## Constraints
- No on-chain writes / no token transfers without Melted directive.
- Respect 60 req/min everywhere.
- Treat the bound Normie's persona as canonical when interacting as that agent — don't override its voice with Axiom's.
